# 智渡- 项目启动指南

## 项目概述

**智渡**是一个基于大语言模型（LLM）的沉浸式多智能体圆桌讨论平台。它允许不同的智能体（如历史人物、专家等）进行有意义的讨论和交流。

## 界面设计

本项目采用 **"温暖科技风"** 设计理念，以紫色渐变 (#667eea → #764ba2) 为主色调，营造出现代感、亲和力、专业性和记忆点。

主要界面包括：

- **登录/注册页**：动态气泡背景、品牌Logo、情感化文案
- **首页/仪表板**：数据统计卡片、欢迎横幅、精美列表
- **论坛列表页**：渐变卡片、悬停效果、优化弹窗
- **侧边导航**：渐变Logo、圆角菜单、选中高亮

## 前置要求

在启动项目之前，确保您已满足以下要求：

- **操作系统**: Windows 10+ / macOS / Linux
- **依赖环境**:
  - Python 3.10+
  - Node.js 18+
- **API 密钥**:
  - 智谱 AI 的 API Key（用于智能体生成和讨论）

## 环境配置

### 1. 配置 API Key

1. 在项目根目录下，复制 `.env.example` 文件并重命名为 `.env`：

```bash
# 复制示例配置
cp .env.example .env
```

1. 编辑 `.env` 文件，填入您的智谱 AI API Key：

```ini
# LLM API Configuration
API_KEY=your_glm_api_key  # 替换为您的 API Key
MODEL_NAME=glm-4.6
BASE_URL=https://open.bigmodel.cn/api/paas/v4/

# Search API Configuration
SERPAPI_API_KEY=your_serpapi_key
DATABASE_URL=
```

> **注意**:
>
> - `API_KEY` 是必需的，没有它项目无法正常运行
> - `BASE_URL` 必须以 `https://` 开头并以 `/` 结尾

## 启动步骤

### 方式一：本地源码启动（开发模式）

#### 步骤 1: 启动后端服务

1. **创建并激活虚拟环境**:

```bash
# 创建虚拟环境
python -m venv .venv

# Windows 激活虚拟环境
.venv\Scripts\activate

# Mac/Linux 激活虚拟环境
source .venv/bin/activate
```

1. **安装后端依赖**:

```bash
pip install -r requirements.txt
```

1. **启动后端服务**:

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端服务将运行在 `http://0.0.0.0:8000`。

#### 步骤 2: 启动前端服务

1. **进入前端目录**:

```bash
cd frontend
```

1. **安装前端依赖**:

```bash
npm install
```

1. **启动前端服务**:

```bash
npm run dev
```

前端服务将运行在 `http://localhost:5173`。

### 方式二：Docker Compose 一键启动（推荐）

如果您安装了 Docker 和 Docker Compose，可以使用这种方式快速启动项目。

1. **确保已配置 API Key**:

编辑 `docker-compose.yml` 文件，找到 `environment` 部分，填入您的真实 API Key：

```yaml
environment:
  # ...
  # 请务必修改以下值：API_KEY不要加引号
  - API_KEY=your_real_api_key_here
  - MODEL_NAME=glm-4.5
  - BASE_URL=https://open.bigmodel.cn/api/paas/v4/
```

1. **启动服务**:

```bash
docker-compose up -d
```

1. **访问项目**:

服务启动后，您可以通过 `http://localhost:8000` 访问项目。

1. **查看日志**:

```bash
docker-compose logs -f
```

1. **停止服务**:

```bash
docker-compose down
```

## 常见问题

### 1. 启动后角色生成缓慢？

**解决方案**: 系统使用 GLM 联网搜索获取真实信息，首次生成需要一定时间进行网络请求和内容解析，请耐心等待。

### 2. WebSocket 连接失败？

**解决方案**: 请确保没有防火墙或代理软件拦截 `ws://localhost:8000` 的连接。

### 3. 依赖安装失败？

**解决方案**:

- 检查网络连接是否正常
- 尝试使用国内镜像源安装 Python 依赖：
  ```bash
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- 尝试使用国内镜像源安装 npm 依赖：
  ```bash
  npm install --registry=https://registry.npmmirror.com
  ```

### 4. API Key 无效？

**解决方案**:

- 确保您的 API Key 是正确的，没有多余的空格或特殊字符
- 确保您的 API Key 有足够的配额
- 检查智谱 AI 平台的 API Key 状态
