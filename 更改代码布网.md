# 更改代码布网指南

本指南详细说明如何在修改源代码并推送到 GitHub 后，更新网页上的内容。适用于当前已部署在腾讯云服务器上的项目。

## 界面设计更新说明

### 设计理念

本项目采用 **"温暖科技风"** 设计理念，以紫色渐变 (#667eea → #764ba2) 为主色调。

### 主要界面文件

- **登录页**: `frontend/src/views/LoginView.vue`
- **注册页**: `frontend/src/views/RegisterView.vue`
- **首页/仪表板**: `frontend/src/views/HomeView.vue`
- **论坛列表页**: `frontend/src/views/ForumListView.vue`
- **布局组件**: `frontend/src/layouts/BasicLayout.vue`
- **全局样式**: `frontend/src/style.css`

### 设计更新后的部署

如果仅修改了界面设计相关文件，按照正常的前端更新流程操作即可：

1. 拉取最新代码
2. 安装前端依赖（如有新增）
3. 构建前端
4. 重启 Nginx（如需要）

## 1. 前提条件

- 已安装 Python 3.10+

## 2. 代码更新流程

### 2.1 本地修改代码

1. **开发前准备**
   ```bash
   # 拉取最新代码，避免冲突
   git pull origin main

   # 查看当前分支，确保在main分支
   git branch
   ```
2. **本地修改与测试**
   - 修改源代码文件
   - **前端测试**：启动开发服务 `npm run dev`，在浏览器中验证修改效果，确认所有功能正常
   - **后端测试**：如果修改了后端代码，启动后端服务 `uvicorn app.main:app --reload`，测试API接口功能
   - **整体测试**：登录、注册、创建论坛、生
     ```shellscript
     git branch
     ```
     成角色等核心功能验证，确保没有bug
3. **提交代码到 GitHub**
   ```bash
   # 查看修改的文件
   git status

   # 添加修改的文件到暂存区
   git add .

   # 提交代码（请写清晰的提交信息）
   git commit -m "feat: 更新登录页logo图标和渐变配色"
   # 提交信息规范：feat(新功能)/fix(修复bug)/docs(文档)/style(样式)/refactor(重构): 具体修改内容

   # 拉取最新代码，避免冲突
   git pull origin main

   # 推送到GitHub
   git push origin main
   ```
4. **提交后检查**
   - 登录GitHub仓库，确认代码已经成功推送
   - 检查提交记录是否正确

> ⚠️ 注意：如果出现代码冲突，先解决冲突后再提交推送

### 2.2 服务器更新流程

#### 2.2.1 部署前准备（必做）

1. **数据备份**
   ```bash
   # 备份数据库
   cp /opt/madf/madf.db /opt/madf/backup/madf_$(date +%Y%m%d_%H%M%S).db

   # 备份前端静态文件
   cp -r /opt/madf/frontend/dist /opt/madf/backup/dist_$(date +%Y%m%d_%H%M%S)
   ```
2. **检查服务器状态**
   ```bash
   # 检查服务运行状态
   sudo systemctl status madf
   sudo systemctl status nginx

   # 检查磁盘空间
   df -h
   ```

#### 2.2.2 分场景更新流程

根据修改内容选择对应的更新流程：

##### 场景A：仅修改前端代码（样式、页面、交互等）

```bash

# 2. 进入项目目录

# 3. 拉取最新代码

# 4. 进入前端目录
cd frontend

# 5. 安装依赖（如果package.json有修改）
npm install

# 6. 构建前端
npm run build

# 7. 检查构建产物
ls -la dist/

# 8. 重启Nginx（可选）
sudo systemctl restart nginx
```

##### 场景B：仅修改后端代码（API、逻辑、数据库等）

```bash
# 1. 登录服务器
ssh root@150.158.166.38

# 2. 进入项目目录
cd /opt/madf

# 3. 拉取最新代码
git pull origin main

# 4. 激活虚拟环境
source venv/bin/activate

# 5. 更新依赖（如果requirements.txt有修改）
pip install -r requirements.txt

# 6. 数据库迁移（如果修改了数据库结构）

sudo systemctl restart madf

# 8. 查看服务状态
sudo systemctl status madf
```

##### 场景C：前后端都有修改

合并场景A和场景B的步骤：

1. 拉取最新代码
2. 构建前端
3. 更新后端依赖
4. 重启后端服务
5. 重启Nginx

#### 2.2.3 验证部署（必做）

1. **基础验证**
   - 访问网站 <https://你的域名，确认页面正常显示>
   - 查看浏览器控制台，确认没有报错
2. **功能验证**
   - 测试登录/注册功能
   - 测试助手仓库、智能体工坊、圆桌论坛等核心功能
   - 测试女娲生成角色功能（如果修改了相关代码）
3. **接口验证**
   ```bash
   # 测试后端API是否正常
   curl http://localhost:8000/api/v1/health

   # 查看后端日志
   journalctl -u madf -f --lines 50
   ```
4. **性能验证**
   - 页面加载速度正常
   - 接口响应时间正常
   - 没有500/404等错误

#### 2.2.4 部署完成

- 确认所有功能正常后，部署完成
- 如果发现问题，参考文档后面的【紧急回滚方案】进行回滚

```bash
# 返回项目根目录

```

如果修改了静态文件路径或 Nginx 配置，需要重启 Nginx：

```bash

```

## 6. 常见问题排查

### 6.1 前端构建失败

**问题**：`npm run build` 失败

**解决方案**：

- 检查 Node.js 版本是否符合要求（18+）
- 检查依赖是否正确安装
- 查看详细错误信息并修复

### 6.2 后端启动失败

**问题**：Uvicorn 启动失败

**解决方案**：

- 检查 Python 依赖是否正确安装
- 检查端口是否被占用
- 查看详细错误日志

### 6.3 网站无法访问

**问题**：出现 404 或 502 错误

**解决方案**：

- 检查 Nginx 配置是否正确
- 检查后端服务是否运行
- 检查安全组配置是否放行端口

### 6.4 数据库连接问题

**问题**：后端无法连接数据库

**解决方案**：

- 检查数据库配置是否正确
- 检查数据库服务是否运行
- 检查数据库文件权限

## 7. 自动化部署方案

### 7.1 使用 GitHub Actions

#### 7.1.1 创建 GitHub Actions 配置文件

在项目根目录创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Tencent Cloud

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Build frontend
        run: |
          cd frontend
          npm install
          npm run build

      - name: Deploy to server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USERNAME }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          script: |
            cd /opt/madf
            git pull origin main
            cd frontend
            rm -rf node_modules package-lock.json
            npm install
            npm run build
            cd ..
            source venv/bin/activate
            pip install -r requirements.txt
            sudo systemctl restart madf
            sudo systemctl restart nginx
```

#### 7.1.2 配置 GitHub Secrets

在 GitHub 仓库设置中添加以下 Secrets：

- `SERVER_HOST`：服务器公网 IP
- `SERVER_USERNAME`：服务器用户名
- `SERVER_SSH_KEY`：服务器 SSH 私钥

### 7.2 使用 Webhook

#### 7.2.1 创建部署脚本

创建 `deploy.sh` 文件：

```bash
#!/bin/bash

# 进入项目目录
cd /opt/madf

# 拉取最新代码
git pull origin main

# 构建前端
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run build
cd ..

# 更新后端依赖
source venv/bin/activate
pip install -r requirements.txt
deactivate

# 重启服务
sudo systemctl restart madf
sudo systemctl restart nginx

# 记录部署日志
echo "Deployed at $(date)" >> deploy.log
```

#### 7.2.2 配置 Webhook

在 GitHub 仓库设置中添加 Webhook，指向服务器的部署脚本。

## 8. 紧急回滚方案

如果部署出现问题，需要回滚到之前的版本：

### 8.1 回滚代码

```bash
# 查看提交历史
git log --oneline

# 回滚到指定版本
git reset --hard <commit_hash>
```

### 8.2 重新构建和部署

```bash
# 重新构建前端
cd frontend
npm run build
cd ..

# 重启服务
sudo systemctl restart madf
sudo systemctl restart nginx
```
